## Existing Dialog and Sheet Infrastructure

- `ThemeDialogsHelper_V2` owns three post-adjacent dynamic menus: user menu, reputation menu, and post menu. The post menu is a `DynamicDialogMenu<ThemeWebCallbacks, IBaseForumPost>` with actions in this order: reply, quote from clipboard, report, edit, delete, copy link, create note, share.
- `DynamicDialogMenu` renders a Material alert dialog with a custom vertical list view. It is not a bottom sheet, but it is the current action menu system used by post actions and WebView link/image context menus.
- `DialogsHelper` uses the same `DynamicDialogMenu` for WebView link/image context menus: open in tab, open in browser, copy link, open image, save image, copy image link.
- `BottomSheetDialog` exists as reusable infrastructure, but UX-01 found no current post action menu using it.

## JS Bridge and Selection Notes

- The topic WebView registers `IThemePresenter` through `ThemeBridgeHandler`; `ThemeJsInterface` exposes post actions to trusted local template JS.
- Inline footer actions currently cover reply, full quote, and post rating votes. Report/edit/delete blocks exist in the template as commented legacy links and are surfaced through the native post menu instead.
- Copy/share/quote selected text are native WebView `ActionMode` actions wired in `ThemeWebController.setupActionModeListener(...)`, then routed through `ThemeBridgeHandler` and JS functions in `theme.js`.
- Post menu actions rely on parsed permission flags from `ThemeParser` and menu filtering in `ThemeDialogsHelper_V2.showPostMenu(...)`.

## Safe Follow-Up Path

1. If UX-02 introduces a cleaner action sheet, start from `ThemeDialogsHelper_V2.showPostMenu(...)` and keep the existing `ThemeWebCallbacks` calls unchanged.
2. Preserve `ThemeViewModel.onPostMenuClick()` edit-form warmup before showing any replacement menu.
3. Keep selected-text actions separate unless the new UI has an explicit selected-text state from WebView.
4. Do not change JS bridge method names or destructive action confirmation behavior without a separate security/behavior task.
