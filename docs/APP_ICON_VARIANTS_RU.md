# Выбор иконки приложения

**Текущее состояние:** три варианта — «Стандартная» (оранжевая P4DA, значение по
умолчанию), «Тёмная» (белая «4» на `#191919`) и «Синяя» (синий градиент на белом).
Две последние собраны из SVG-экспорта `~/Downloads/ProPDA-icons-svg/` и фон у них
фиксированный: каждая сама себе выбор, день/ночь их не переключает.

Настройки → «Внешний вид» → **Иконка приложения** показывает сетку доступных
иконок. Выбранная запоминается в `main.app_icon` и применяется:

* к ярлыку на рабочем столе и в списке приложений (`activity-alias`);
* к иконке в «Недавних» (её берёт тот же псевдоним);
* к экрану загрузки (splash-тема варианта);
* к большой иконке уведомлений о загрузках.

Иконку в системных «Настройках → Приложения» сменить нельзя: она берётся из
`<application android:icon>`, а это статический ресурс, недоступный для
переключения в рантайме. Это ограничение Android, не недоделка.

## Как добавить свою иконку

Нужны мастера 1:1 (PNG, желательно ≥1024 px):

| Мастер | Обязателен | Чем может быть |
| --- | --- | --- |
| светлый | да | плоский квадрат с фоном **или** рисунок на прозрачном фоне |
| тёмный | нет (по умолчанию = светлый) | то же самое |
| монохромный | нет (по умолчанию — силуэт светлого) | белая маска на прозрачном фоне |

```bash
python3 design/app-icon/add_alt_icon.py \
  --id my_icon --title-ru "Моя иконка" --title-en "My icon" \
  --light path/to/light.png --dark path/to/dark.png --mono path/to/mono.png
```

Скрипт сам:

1. геометрия по `orange-p4da-adaptive/SPEC.md`: убирает у мастера залитый фон (по
   цвету углов), приводит его к квадрату, центрирует знак, сжимает **весь квадрат
   мастера до 68%** холста и центрирует итоговую маску ещё раз. Масштаб считается
   от квадрата мастера, а не от обрезанного контура — иначе широкие знаки лезут
   за круглую маску. Одна и та же геометрия применяется к светлому, AMOLED и
   монохромному слою, поэтому размеры совпадают; расхождение противоположных
   полей проверяется и валит сборку при разнице больше 2 px;
2. кладёт ресурсы (`drawable*/`, `mipmap-anydpi-v26/`), цвета фона в
   `values/colors.xml` и `values-night/colors.xml`, названия в `strings.xml`
   (ru + en), splash-тему в `values/styles.xml`;
3. дописывает `activity-alias` в `AndroidManifest.xml` и запись в
   `AppIcons.variants`;
4. кладёт лист сверки в `design/app-icon/alt-icons/<id>-review.png`: все четыре
   слоя (светлый, AMOLED, Monet, monochrome) под тремя масками лаунчера — круг,
   сквиркл и скруглённый квадрат. Проверять обязательно: круг режет углы сильнее
   остальных, и обрезку видно только на нём.

После этого достаточно пересобрать приложение — новая иконка появится в пикере.
Повторный запуск с тем же `--id` перезаписывает вариант на месте.

### Полезные ключи

* `--as-is` — мастера уже подготовлены как слои adaptive-иконки (нужный масштаб
  и центровка). Скрипт тогда ничего не вырезает и не масштабирует. Так
  подключена «P4DA оранжевая»: её геометрию считает отдельный утверждённый
  генератор `generate_orange_p4da_adaptive.py` (см. ниже).
* `--scale 0.62` — знак всё равно кажется крупным или, наоборот, теряется.
  По умолчанию `0.68` (значение из SPEC.md). Ориентир: видимая зона
  adaptive-иконки — 0.667 холста, и то по квадрату, круглая маска режет углы.
* Векторный экспорт из Figma сначала растеризуйте:
  `python3 design/app-icon/render_flat_svg.py in.svg out.png` — он отдаёт знак на
  прозрачном фоне и печатает цвет фона из `<rect>` для `--light-bg`/`--dark-bg`.
  Путь, залитый цветом фона, трактуется как вырез (так Figma рисует выемки), а
  подпути внутри одного `d` складываются по even-odd, поэтому внутренний контур
  становится дыркой. QuickLook (`qlmanage`) для этого не годится: он добавляет по
  краю полоску незакрытого `viewBox`, из-за которой не определяется фон.
* `--light-bg "#F7F5F0"` / `--dark-bg "#101114"` — задать фон вручную. Нужно,
  когда мастер уже на прозрачном фоне (угадать неоткуда) или углы не отражают
  реальный фон.

Пример подключения арта из чужого генератора (так собрана оранжевая):

```bash
python3 - <<'PY'
import importlib.util, sys
from pathlib import Path
spec = importlib.util.spec_from_file_location(
    "gen", "design/app-icon/generate_orange_p4da_adaptive.py")
gen = importlib.util.module_from_spec(spec); sys.modules["gen"] = gen
spec.loader.exec_module(gen)
out = Path("/tmp/orange"); out.mkdir(exist_ok=True)
gen.approved(gen.LIGHT_MASTER).save(out / "light.png")
gen.approved(gen.AMOLED_MASTER).save(out / "dark.png")
gen.themed_mask().save(out / "mono.png")
PY

python3 design/app-icon/add_alt_icon.py \
  --id p4da_orange --title-ru "P4DA оранжевая" --title-en "P4DA orange" \
  --as-is --light /tmp/orange/light.png --dark /tmp/orange/dark.png \
  --mono /tmp/orange/mono.png
```

### Чего делать нельзя

* **Менять `--id` и имя псевдонима у уже выпущенной иконки.** Id хранится в
  настройках пользователя, а имя псевдонима — в ярлыках на рабочем столе:
  переименование обнулит выбор и убьёт ярлыки.
* **Возвращать фильтр `MAIN`/`LAUNCHER` в саму `MainActivity`.** Ярлык даёт
  только включённый `activity-alias`; иначе рядом появится второй ярлык.
* **Забывать `android.app.shortcuts` в новом псевдониме.** Система читает
  статические ярлыки только у компонента с `MAIN`/`LAUNCHER` — без этого
  «Ответы», «Личное» и остальные пропадут из длинного тапа по иконке.

## Как это устроено

* `AppIcons` (`common/appicon/AppIcons.kt`) — реестр вариантов: id, имя
  псевдонима, название, ресурс значка, splash-тема.
* `AppIconManager` — включает выбранный псевдоним и гасит остальные. Порядок
  важен: сначала включаем новый, потом выключаем старые, иначе лаунчер на
  мгновение видит пакет без ярлыка.
* Применение отложено до ухода приложения в фон (`App.AppLifecycleObserver`).
  Выключение псевдонима, с которого запущена текущая задача, рушит её даже с
  `DONT_KILL_APP` — при мгновенном применении пользователь вылетал из настроек
  на рабочий стол. Поэтому пикер только пишет настройку, а состояние
  компонентов приводится к ней при сворачивании. Об этом же говорит подсказка
  «Иконка сменится, когда вы свернёте приложение».
* `AppIconManager.ensureLauncherPresent` вызывается из `App.onCreate` при любом
  старте процесса и только ВКЛЮЧАЕТ нужный псевдоним, никого не выключая
  (поэтому задаче ничего не грозит). Это защита от состояния «приложение совсем
  без ярлыка» — см. следующий раздел.
* Splash: `MainActivity.applySelectedIconSplashTheme()`. На Android 12+ splash
  рисует система по манифестной теме, поэтому используется
  `SplashScreen.setSplashScreenTheme` — он действует со следующего запуска, что
  совпадает с отложенной моделью выше. До 12 splash рисует androidx уже внутри
  активити, там достаточно `setTheme` до `installSplashScreen()`.
* День/ночь работает сам: `mipmap-anydpi-v26/ic_launcher_<id>.xml` ссылается на
  слои, у которых есть `-night` версии (`drawable-night-nodpi/…` и цвет фона в
  `values-night/colors.xml`). Отдельного «тёмного варианта» в пикере нет.

## Можно ли грузить иконки архивом из приложения?

Нет — для ярлыка это невозможно в принципе. `activity-alias android:icon` —
скомпилированный id ресурса; API, позволяющего подставить лаунчеру произвольный
файл, в Android нет ни на одной версии. Поэтому «комплект иконки» — это всегда
шаг сборки: положить мастера, прогнать `add_alt_icon.py`, собрать APK.

Что технически реализуемо, если очень нужен пользовательский рисунок:

* **Закреплённый ярлык** (`ShortcutManager.requestPinShortcut` +
  `Icon.createWithBitmap`) — отдельный ярлык на рабочем столе с любой картинкой,
  запускающий приложение. Иконка самого приложения в списке приложений при этом
  не меняется, и ярлык нельзя обновить без повторного запроса.
* **Уведомления** — большая иконка это обычный `Bitmap`, её можно рисовать из
  файла.
* **Splash** — только на Android < 12 (там значок рисует библиотека внутри
  активити). На 12+ значок берёт система из темы, до старта процесса.

## Удалить вариант

Обратной команды у скрипта нет — уберите вручную:

1. запись из `AppIcons.variants`;
2. `activity-alias` из манифеста;
3. splash-тему `Theme.ForPDA.Splash.<Camel>` из `values/styles.xml`;
4. ресурсы `ic_launcher_<id>*`, цвет `ic_launcher_<id>_background`, строку
   `app_icon_<id>`.

**Ловушка, из-за которой удалять вариант опаснее, чем добавлять.** Состояние
компонентов живёт в системе и переживает обновление приложения. Если убрать
вариант, который пользователь уже выбрал, то после апдейта его псевдоним из
манифеста исчезает, а псевдоним по умолчанию остаётся ЯВНО выключенным — у
приложения не остаётся ни одного включённого `MAIN`/`LAUNCHER`-компонента,
то есть ярлыка нет вообще и запустить приложение, чтобы это исправить, нечем.
Проверено живьём на эмуляторе.

Спасает `ensureLauncherPresent` из `App.onCreate`: любой старт процесса —
фоновый воркер, уведомление, deep-link, ярлык другого приложения — возвращает
ярлык на место (`AppIcons.byId` для неизвестного id отдаёт вариант по
умолчанию). Не удаляйте этот вызов и не переносите его в `MainActivity`: до
активити в такой ситуации может быть не добраться.
